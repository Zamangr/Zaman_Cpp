# Пояснение к проекту (для преподавателя)

Этот документ кратко объясняет, где в проекте находится каждая часть логики и какие функции за что отвечают.

---

## 1. Общая структура

- `src/main.cpp` — точка входа, запуск бенчмарка, вывод таблицы результатов, экспорт CSV.
- `include/Schedule.h` — структуры данных: `Match`, `Tour`, `Schedule`.
- `include/IScheduler.h` — абстрактный интерфейс для всех алгоритмов.
- `include/Validator.h`, `src/Validator.cpp` — проверка корректности расписания (жёсткие ограничения H1, H2).
- `include/Metrics.h`, `src/Metrics.cpp` — подсчёт числа нарушений (breaks), метрики качества Q.
- `include/BergerScheduler.h`, `src/BergerScheduler.cpp` — алгоритм Бергера (круговой метод).
- `include/GreedyScheduler.h`, `src/GreedyScheduler.cpp` — рандомизированный жадный алгоритм.
- `include/BacktrackScheduler.h`, `src/BacktrackScheduler.cpp` — алгоритм с возвратом (backtracking).
- `include/GeneticScheduler.h`, `src/GeneticScheduler.cpp` — генетический алгоритм.

---

## 2. Структуры данных

Файл: `include/Schedule.h`

- `struct Match` — один матч:
  * `home` — индекс команды-хозяина поля;
  * `away` — индекс команды-гостя;
  * `unordered_pair()` — пара без учёта порядка (для проверки уникальности).

- `using Tour = std::vector<Match>` — тур: набор из `n/2` одновременных матчей.

- `class Schedule` — полное расписание турнира:
  * `n_teams()` — число команд;
  * `n_tours()` — число туров (равно `2*(n-1)` для двухкругового);
  * `tour(r)` — тур с индексом r;
  * `add_tour(t)` — добавить тур;
  * `home_pattern(team)` — вектор статусов дома/в гостях для команды.

- `class IScheduler` — интерфейс:
  * `virtual Schedule solve(int n) = 0` — построить расписание для n команд.

## 3. Алгоритм Бергера

Файл: `src/BergerScheduler.cpp`

Реализует классический «круговой метод» (circle method, 1873).

- Команда `n-1` фиксируется. Остальные `n-1` команд размещаются в массиве `arr`.
- На каждом туре пары формируются как соединения по кругу: позиция `i` ↔ позиция `n-1-i`.
- После каждого тура массив `arr` циклически сдвигается:
  * новый `arr[1]` = старый `arr[n-1]`;
  * остальные сдвигаются на одну позицию вправо.
- Второй круг строится зеркально: для каждого матча `(h, a)` добавляется `(a, h)`.
- Сложность: **O(n²)** по времени и памяти.

## 4. Жадный алгоритм

Файл: `src/GreedyScheduler.cpp`

Рандомизированный жадный алгоритм с перезапусками.

- Расписание строится тур за туром.
- Для каждого тура вызывается `buildTour()`:
  * команды перебираются в случайном порядке;
  * для каждой непарованной команды ищется партнёр с минимальным штрафом `calcPenalty()`;
  * штраф учитывает нарушение чередования домашних/выездных матчей.
- Эвристика назначения хозяина: команда с большим числом домашних матчей получает роль гостя.
- При тупике (невозможно завершить тур) — перезапуск с новым зерном (`MAX_RESTARTS = 8`).
- Из нескольких прогонов выбирается расписание с наименьшим числом нарушений.
- Сложность: **O(R · n² · log n)**, где R — число перезапусков.

---

## 5. Алгоритм с возвратом (Backtracking)

Файл: `src/BacktrackScheduler.cpp`

Точный рекурсивный поиск с ветвлением и отсечениями.

- Расписание строится рекурсивно тур за туром.
- На каждом шаге перебираются все возможные совершенные паросочетания.
- **Отсечение (branch-and-bound):** если текущее число нарушений `current_breaks >= best_breaks` — ветвь отбрасывается.
- **Лимит времени:** конструктор принимает `time_limit_ms`; при превышении возвращается лучшее найденное решение.
- При `n = 6` достигается теоретический оптимум `B_min = n - 2 = 4` (де Верра, 1981).
- Сложность в худшем случае: **O((n/2)!^{2(n-1)})** — экспоненциальная.

## 6. Генетический алгоритм

Файл: `src/GeneticScheduler.cpp`

Метаэвристика на основе эволюционных операторов.

### 6.1 Инициализация популяции
- 1 особь — расписание Бергера;
- несколько особей — жадный алгоритм с разными зёрнами;
- остальные — мутации расписания Бергера.

### 6.2 Функция приспособленности
- `count_breaks(s)` — суммарное число нарушений чередования сторон поля.

### 6.3 Селекция
- Турнирный отбор: из `k = 3` случайных особей выбирается лучшая.

### 6.4 Скрещивание (crossover)
- `crossover_uniform()` — для каждой пары команд случайно выбирается тур от одного из родителей.
- Если корректное расписание не получается — используется копия первого родителя (fallback).

### 6.5 Мутации (сохраняют корректность DRR)
- `mutate_swap_tours()` — переставляет два случайных тура местами. Жёсткие ограничения сохраняются.
- `mutate_flip_mirror_pair()` — инвертирует стороны поля для случайной пары команд в обоих матчах.

### 6.6 Параметры
- Размер популяции: `P = 50`
- Число поколений: `G = 200`
- Вероятность мутации: `p = 0.15`
- Элитизм: 10% лучших особей переносятся без изменений.
- Сложность: **O(G · P · n²)**.

## 7. Подсистема валидации и метрик

Файл: `src/Validator.cpp`, `src/Metrics.cpp`

- `Validator::validate(schedule)` — проверяет:
  * каждая упорядоченная пара команд встречается ровно 1 раз (H1);
  * в каждом туре каждая команда участвует ровно 1 раз (H2);
  * число туров равно `2*(n-1)`.
- `Metrics::countBreaks(schedule)` — суммарное число нарушений чередования.
- `Metrics::maxStreak(schedule)` — максимальная длина серии (домашних или выездных) подряд.
- `Metrics::quality(schedule)` — метрика качества `Q ∈ [0, 1]`, где `Q = 1` — идеальное чередование.

## 8. Точка входа (main.cpp)

Файл: `src/main.cpp`

- Запускает бенчмарк для всех 4 алгоритмов при `n ∈ {6, 8, 10, 12}`.
- Выводит сводную таблицу в консоль: время, нарушения, качество, корректность.
- Экспортирует результаты в `results/benchmark.csv`.
- Дополнительно: 30 запусков генетического алгоритма при `n = 8` для оценки стабильности → `results/ga_stability.csv`.

## 9. Как быстро проверить работу

### Сборка через g++ (macOS / Linux)
```bash
g++ -std=c++20 -O2 -Iinclude \
  src/Schedule.cpp src/Validator.cpp src/Metrics.cpp \
  src/BergerScheduler.cpp src/GreedyScheduler.cpp \
  src/BacktrackScheduler.cpp src/GeneticScheduler.cpp \
  src/main.cpp -o benchmark
```

### Запуск
```bash
./benchmark
```

### Ожидаемый вывод
```

Greedy        6    0.285      8        0.852     YES
```
